Project Overview

CALMA es una aplicación web interactiva diseñada para ser un refugio digital y una herramienta proactiva para el autocuidado y la gestión de la salud emocional en el día a día. La aplicación propone cuidar un capibara como reflejo del autocuidado personal, integrando un diario emocional inteligente y un chatbot de bienestar.

Core Features

  • Journal Emocional Inteligente: Escribe tus pensamientos y deja que nuestra IA analice el sentimiento para colorear tu calendario emocional
  • Retos Diarios Adaptativos: Basado en tus entradas del diario, la aplicación te sugiere tareas personalizadas para ayudarte a gestionar tus emociones
  • Capybara Sensei: Un chatbot conversacional (impulsado por un LLM) que te ofrece un espacio seguro para hablar, escuchar y recibir consejos basados en técnicas de bienestar
  • Gamificación Sutil: El concepto de cuidar de tu mascota (el Capibara) a través de tu propio autocuidado (completando retos) crea un ciclo de motivación positivo
  • Interfaz Limpia y Reactiva: Construida con tecnologías web modernas para una experiencia de usuario fluida y relajante

System Demonstration

System Architecture

La arquitectura de CALMA está diseñada para ser modular y eficiente, permitiendo una interacción fluida entre el frontend, una API mock para la persistencia de datos y un servicio externo de IA para el análisis de sentimientos.

  • Frontend: React con Vite para una interfaz rápida y potente
  • Backend Mock: Node.js & Express.js para API RESTful simulada
  • Base de Datos: Archivo JSON como base de datos NoSQL local
  • IA: Llama 3B Instruct para análisis de sentimiento y generación de tareas

Technical Challenges

Varios desafíos técnicos complejos fueron abordados durante el desarrollo, incluyendo la gamificación ética, análisis de sentimiento preciso, privacidad de datos y experiencia de usuario fluida.

  • Gamificación Ética: Equilibrar la motivación del usuario con el respeto por la salud mental
  • Análisis de Sentimiento Preciso: Garantizar que el modelo de IA interprete correctamente las emociones del usuario sin sesgos
  • Privacidad de Datos: Proteger la información emocional sensible del usuario mientras se mantiene la funcionalidad
  • Experiencia de Usuario Fluida: Crear una interfaz que sea tanto atractiva como no abrumadora para usuarios en estados emocionales vulnerables

Technical Roadmap

Las mejoras técnicas planificadas se enfocan en expandir las capacidades de la aplicación, mejorar la experiencia del usuario y extender la integración del sistema con fuentes de datos adicionales.

  • Integración con Wearables para obtener datos biométricos que enriquezcan el análisis emocional
  • Marketplace de Bienestar con más contenido y herramientas personalizadas
  • Comunidad de Usuarios para crear espacios seguros de apoyo mutuo
  • Mejoras en el modelo de IA para análisis de sentimiento más preciso
